# Entities

`entities`는 앱에서 반복해서 쓰는 도메인 단위 타입, 상수, 상태를 둡니다.

## user

현재 로그인/게스트 사용자를 앱 전역에서 표시하기 위한 클라이언트 상태입니다.

- Supabase Auth의 `auth.users` row 자체가 아닙니다.
- `profiles`, `user_activity_stats`, `user_wallets`에서 가져온 요약값을 화면에서 공유합니다.
- 레이아웃, 내정보, 로비처럼 여러 화면에서 공통으로 쓰는 사용자 표시 상태를 다룹니다.

## profile

내정보 페이지에서 보여주는 상세 프로필 데이터입니다.

- 이름, 자기소개, 가입일, 퀴즈 통계, 활동 통계처럼 내정보 화면에 필요한 데이터를 다룹니다.
- 전역 상태가 아니라 프로필 화면의 표시 모델에 가깝습니다.

## daily-quest

로비에서 보여주는 데일리 퀘스트의 표시 모델입니다.

- 퀘스트 제목, 진행도, 보상, 완료/수령 상태를 다룹니다.
- 하루 단위 미션 UI에서 반복해서 쓰는 타입을 둡니다.
- 실제 보상 지급이나 수령 처리는 서버 로직과 분리해서 다룹니다.

## mini-quiz

미니 퀴즈의 챕터와 스테이지 콘텐츠 모델입니다.

- 챕터, 스테이지, 잠금/완료 상태, 별 개수 같은 학습 진행 표시값을 다룹니다.
- 퀴즈 진행 화면과 학습 화면에서 공유하는 콘텐츠 상수를 둡니다.
- 문제 풀이 시도나 채점 결과 저장은 Supabase의 attempt/progress 테이블과 분리해서 다룹니다.
