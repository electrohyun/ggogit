# 데일리 퀴즈 Lazy 생성 방식

## 결론

데일리 퀴즈는 매일 00시에 Supabase가 자동 생성하지 않는다.
대신 사용자가 오늘의 퀴즈 화면에 처음 들어오는 순간 서버가 오늘 문제 세트를
조회하고, 없으면 즉시 생성한다.

## 날짜 기준

- 기준 날짜는 서버에서만 계산한다.
- 한국 서비스 기준으로 `Asia/Seoul` 날짜를 사용한다.
- 클라이언트는 오늘 날짜나 갱신 가능 여부를 직접 판단하지 않는다.

```sql
(now() at time zone 'Asia/Seoul')::date
```

## 흐름

```txt
00:00 KST
- 날짜만 바뀐다.
- DB에 오늘 문제 세트가 아직 없을 수 있다.

첫 사용자 접속
- 서버가 오늘 날짜의 daily_git_quiz_sets row를 찾는다.
- 없으면 미니 퀴즈 문제 풀에서 5문제를 골라 생성한다.
- 생성된 5문제를 사용자에게 반환한다.

이후 사용자 접속
- 이미 생성된 오늘 문제 세트를 그대로 반환한다.
```

## 제출 기준

- 로그인 사용자가 오늘 첫 completed 기록을 저장하면 랭킹에 반영한다.
- 이때 50콩을 지급하고 스트릭을 갱신한다.
- 같은 날 재도전은 결과만 반환하고 랭킹, 콩, 스트릭에 반영하지 않는다.
- 게스트는 결과만 반환하고 DB에 저장하지 않는다.

## 자정 경계

플레이 중 자정이 지나 제출하면 서버 기준 오늘 문제 세트와 다를 수 있다.
v0.5에서는 이전 날짜 제출을 보존하지 않고, 새 오늘의 퀴즈를 다시 시작하게 한다.
